Core Engine

Real-time integrity monitoring.

The foundation. Every other feature builds on top of a rock-solid, multithreaded integrity engine.

🔍
Event-Driven File Monitoring
Replaced polling with watchdog event hooks. Detects create, modify, delete, rename — including editor "atomic save" patterns. Debounce timer (2s) prevents false positives during large file writes.
⚡
Multithreaded Hash Engine
Parallel hashing using ThreadPoolExecutor across all CPU cores. Benchmark: 119 GB in 65s (1,830 MB/s) on NVMe. Python is no longer the bottleneck — hardware I/O is the limit.
🔒
HMAC-Signed Audit Logs
Every log line is signed with an HMAC-SHA256 signature. Tamper detection operates at write time. Log rotation preserves matching .sig backup files. Secure viewer decrypts in RAM — files stay encrypted on disk.
🏗
State Hashing
Master hash = SHA-256(content_hash | attributes | mtime). Detects property-only changes — hidden bit, archive bit, modification timestamps. Hidden file tracking via os.stat().st_file_attributes.
🔄
Rename Detection
The on_moved handler distinguishes true renames from editor atomic-save patterns (write-new → rename-old → rename-new). Zero false positives from VS Code, PyCharm, or Notepad++.
⚙️
External Config + Hot Reload
All settings in config.json — watch folder, secret key, webhook URL, intervals. Runtime configuration reload without restart. Separation of core engine from interface layers.
Active Defense

Respond, not just detect.

Automated countermeasures that trigger without any human input — from vault restoration to OS-level lockdown.

PRO only
Auto-Heal Vault
Selectively backs up files <10 MB with allowed extensions. Hidden, obfuscated vault path (sha256(path).enc filenames). Instant restoration on modification or deletion. Infinite-loop prevention via debounce + content-hash comparison.
🛑
Ransomware Killswitch
Burst detection: ≥5 file events in 10 seconds. OS-level lockdown via Windows icacls — strips Write/Delete for Everyone at NTFS kernel level. Parallelizes ransomware encryption attempts at kernel level.
🪤
Honeypot Tripwire
secret_passwords.txt monitored separately. Any access — read, write, or attribute change — instantly detonates the killswitch and sends a CRITICAL alert with a forensic snapshot.
🧠
Process Attribution
Every file event records which process caused it. Attribution runs fully asynchronously — events log instantly, process info appears within seconds. One bulk OS call per cycle — GIL hold reduced from ~200ms to under 1ms.
⚠️
LOLBin Detection
25+ known-malicious Windows binaries identified: PowerShell, certutil, mshta, wscript, cscript, regsvr32, and more. Trusted-parent whitelist suppresses false positives from VS Code, PyCharm, Terminal — zero alert fatigue from dev tools.
🗂
System Path Protection
Monitoring expanded to Windows persistence locations: startup folders (HKCU + All Users), scheduled tasks directory, and hosts file. Files created or modified in startup folders trigger CRITICAL severity regardless of file type.
📋
Registry Persistence Monitor
14 Windows Registry keys used by malware families monitored via RegNotifyChangeKeyValue() — a kernel API that uses zero CPU when idle. New Run keys, IFEO debugger hijacks, AppInit DLL modifications, Winlogon changes all trigger CRITICAL alerts.
🔌
USB Device Control
Windows Registry StorageDevicePolicies enforcement. Blocks unauthorized USB write access across the entire device. Policy toggled remotely via C2 tenant settings or locally via the desktop agent.
📸
AES Forensic Snapshots
Auto-generated on every CRITICAL event. Captures system state, disk info, running process list, and last 15 log lines. Encrypted with AES-256 (Fernet) — .dat files unreadable without the hardware-bound KEK.
